html

  </div>

  <div class="center-buttons">
    <img src="assets/tutorial.png" alt="Tutorial" onclick="location.href='home.html'" />
    <img src="assets/iniciar.png" alt="Iniciar" onclick="location.href='servicos.html'" />
    <img src="assets/escolha.png" alt="Escolha um nível" onclick="location.href='informacoes.html'" />

</body>

javascript
document.getElementById("menu-button").addEventListener("click", () => {
  const menu = document.getElementById("menu-list");

  menu.style.display = menu.classList.contains("hidden") ? "none" : "block";
});

🚀 Como postar seu site no GitHub e ativar o GitHub Pages

✅ 1. Crie um repositório

  1. Acesse

  2. Clique no botão "New repository" (canto superior esquerdo)

  3. Dê um nome ao repositório, por exemplo: meu-site-candyland

  4. Marque como Public

  5. Clique em "Create repository"

📂 2. Envie os arquivos do seu site

  1. No seu computador, crie uma pasta chamada meu-site-candyland

  2. Coloque todos os arquivos do projeto dentro dela:

    • index.html, style.css, script.js

    • As páginas: home.html, contato.html, etc.

    • A pasta assets/ com suas imagens

  3. Clique com o botão direito na pasta e selecione "Enviar para > Pasta compactada (.zip)"

  4. No GitHub, vá até o repositório que você criou

  5. Clique em "Add file > Upload files"

  6. Arraste os arquivos da pasta extraída (não o .zip, mas os arquivos de dentro)

  7. Clique em "Commit changes"

🌐 3. Ative o GitHub Pages

  1. No repositório, clique em "Settings"

  2. Role até a seção "Pages"

  3. Em "Source", selecione a branch main e a pasta /root

  4. Clique em "Save"

  5. O GitHub vai gerar um link como: https://seuusuario.github.io/meu-site-candyland

mas e a primeira pasta que precisa, que tem o arquivo index?

O Copilot disse

Boa pergunta, Duda! 💖 A “primeira pasta” que você está falando é a pasta principal do projeto, onde ficam todos os arquivos do seu site — incluindo o index.html, que é a página inicial.

No GitHub, você não precisa criar essa pasta manualmente. Quando você envia os arquivos diretamente para o repositório, o GitHub já considera o repositório como essa pasta principal.

✅ Como funciona na prática

  1. Você cria um repositório chamado meu-site-candyland

  2. Dentro dele, você envia os arquivos diretamente:

    • index.html

    • style.css

    • script.js

    • As outras páginas (home.html, etc.)

    • A pasta assets/ com suas imagens

O GitHub trata esse repositório como a “pasta raiz” do seu site. O index.html precisa estar diretamente nessa raiz, não dentro de outra pasta.

💡 Dica importante

  • Não coloque o index.html dentro de uma subpasta como site/ ou src/

  • Ele precisa estar direto no repositório, para que o GitHub Pages funcione corretamente
